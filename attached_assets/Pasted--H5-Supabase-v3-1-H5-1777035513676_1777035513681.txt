## 一、项目概述与核心意图

### 1.1 产品定义

开发一款精力管理与情绪陪伴 H5 移动应用，产品名称为**小鱼干**。

**核心理念：** 帮助低精力人群（职场女性、宝妈、经期女性、ADHD 患者）通过极简操作记录精力波动，并由 AI 驱动的电子宠物（小猫）提供温暖陪伴、微行动建议和“减法许可”。

**产品定位（开发者必须理解并贯穿全程遵守）：**

| 不是什么 | 是什么 |
|---------|--------|
| 效率工具 | 允许摆烂的陪伴工具 |
| 数据可视化应用 | 自然语言关怀产品 |
| 强制记录工具 | 最低成本捕捉精力状态的工具 |

**核心价值（任何功能的设计必须围绕以下三条）：**
1. **零进入成本**：打开即用，无教程、无前置填表
2. **无惩罚机制**：不显示断更天数，不发催促通知
3. **温暖陪伴**：电子宠物常驻，低精力时主动关怀

### 1.2 目标用户画像

| 用户类型 | 年龄 | 核心诉求 | 设计关键词 |
|---------|------|---------|-----------|
| 职场女性 | 25-40岁 | 快速记录，被理解而非被督促 | 快、轻、不催 |
| 宝妈 | 25-45岁 | 简单情绪出口 + 允许休息的心理支持 | 简单、包容 |
| 经期女性 | 全年龄段 | 记录周期性精力变化 + 针对性回血建议 | 周期感知 |
| ADHD/低精力人群 | 全年龄段 | 极简界面、大按钮、有趣激励而非KPI | 极简、有趣 |

## 二、全局约束（开发过程中始终适用）

> ⚠️ 以下规则优先级最高，任何功能实现若与之冲突，必须以本节为准。

1. **精力值范围约束**：精力值为 `[0, 10]` 的浮点数，步进 `0.5`。`currentEnergy` 的计算结果必须通过 `clamp(value, 0, 10)` 约束，绝不允许超出此范围。
2. **气泡唯一性约束**：任何时刻屏幕上最多同时显示 **1个** 猫咪气泡。新气泡出现前必须关闭现有气泡。使用全局 `activeBubble` 状态锁实现，禁止堆叠。
3. **触发冲突优先级**：当多个触发条件同时满足时，按 `严重透支预警 > 慢性疲劳期 > 盲盒回血` 的优先级只触发 **最高级别的一个**，其余静默排队，等本次流程结束后才可触发。
4. **无惩罚机制**：代码中禁止出现断更天数计算、催促文案、任何表示“你XX天没来了”的逻辑。
5. **单击静默原则**：点击 +/- 按钮的**单击**操作不得触发任何气泡弹窗，仅做动画反馈。
6. **数据确定性**：AI 生成内容（日报、帮助回答）必须严格基于传入的真实数据，禁止编造未发生的事件。提示词必须在 system prompt 中明确写明此约束，不能只写在 user prompt 里。
7. **离线优先**：精力记录、当日初始值等核心状态必须先写入本地缓存（`localStorage`），再异步同步云端。网络异常时不影响主流程。
8. **首次与复访区分**：每次页面 `onShow` 时，必须区分「今日首次打开」和「今日已打开过」两种状态，初始化逻辑不同（见功能 1 详细说明）。

#### 1.4 核心计算公式（v3.1 最终版）

```
S_initial = round(((B × 0.6) + (O_weight × 0.4 × 10)) × P_cycle, 0.5)
```

- **B**：基础体质分值（4, 7, 10）
- **O_weight**：负荷保留率（1.0, 0.7, 0.6），乘以10后与 B 量级对齐
- **P_cycle**：周期阶段系数（见下表）
- 结果自动通过 `clamp(value, 0, 10)` 限制范围，且以0.5为步进四舍五入。

**周期阶段系数表：**

| 周期阶段 | 判定条件（D = 当日距上次月经起始的天数 mod T） | 系数 P_cycle |
|---------|---------------------------------------------|-------------|
| 月经期 | 0 ≤ D < M | 0.8 |
| 卵泡期 | M ≤ D < (T/2 - 2) | 1.1 |
| 排卵期 | (T/2 - 2) ≤ D < (T/2 + 2) | 1.2 |
| 黄体期 | (T/2 + 2) ≤ D < T | 0.7 |
| 无数据（已跳过） | skip_cycle = true | 1.0 |

#### 1.6 首页初始化逻辑（关键，防止状态丢失）

```javascript
/**
 * 首页 onShow 时执行的初始化逻辑
 * 必须严格区分：今日首次打开 vs 今日已打开过 vs 历史用户重新打开
 */
function onHomeShow() {
  const today = getTodayDateString(); // 格式：'YYYY-MM-DD'
  const lastOpenDate = localStorage.getItem('last_open_date');
  const todayInitial = localStorage.getItem(`energy_initial_${today}`);

  if (lastOpenDate === today && todayInitial !== null) {
    // 情况1：今日已打开过 → 读取当日最新精力值，不重置
    const currentEnergy = parseFloat(localStorage.getItem(`energy_current_${today}`) ?? todayInitial);
    renderHome(currentEnergy);

  } else {
    // 情况2：今日首次打开（新的一天）→ 重新计算今日初始值
    const baseline = JSON.parse(localStorage.getItem('user_baseline'));
    if (!baseline) {
      // 情况3：完全新用户，无基线数据 → 触发引导流程
      navigateToOnboarding();
      return;
    }
    const { S_initial, phase } = calculateTodayEnergy(baseline);
    localStorage.setItem(`energy_initial_${today}`, S_initial);
    localStorage.setItem(`energy_current_${today}`, S_initial);
    localStorage.setItem('last_open_date', today);
    renderHome(S_initial, phase);
  }
}

/**
 * 计算今日精力初始值（基于基线数据，重新判定当前周期）
 * 每天首次打开时调用，以当天日期重新判定周期阶段
 */
function calculateTodayEnergy(baseline) {
  const { base_energy: B, daily_load: O_weight, last_period_start,
          cycle_days: T, period_days: M, skip_cycle } = baseline;

  let P_cycle = 1.0, phase = '未追踪';

  if (!skip_cycle && last_period_start) {
    const today = new Date();
    const lastStart = new Date(last_period_start);
    // 防止日期异常（lastStart 为未来日期）
    if (lastStart > today) {
      console.warn('last_period_start 为未来日期，使用默认 P_cycle=1.0');
    } else {
      const daysDiff = Math.floor((today - lastStart) / (1000 * 60 * 60 * 24));
      const D = daysDiff % T;

      if (D < M) { P_cycle = 0.8; phase = '月经期'; }
      else if (D < T / 2 - 2) { P_cycle = 1.1; phase = '卵泡期'; }
      else if (D < T / 2 + 2) { P_cycle = 1.2; phase = '排卵期'; }
      else { P_cycle = 0.7; phase = '黄体期'; }
    }
  }

  // 核心公式：S_initial = round(((B × 0.6) + (O_weight × 0.4 × 10)) × P_cycle, 0.5)
  const raw = (B * 0.6 + O_weight * 0.4 * 10) * P_cycle;
  const S_initial = clamp(roundToHalf(raw), 0, 10);

  return { S_initial, phase, P_cycle };
}

/** 精确到0.5的四舍五入 */
function roundToHalf(value) {
  return Math.round(value * 2) / 2;
}

/** 区间限制 */
function clamp(value, min, max) {
  return Math.min(Math.max(value, min), max);
}
```

#### 2.2 +/- 按钮交互（防竞态设计）

```javascript
/**
 * 按钮状态机：防止单击与长按逻辑互相污染
 * 状态：idle → pressing → (longpress_triggered | click_triggered) → idle
 */
class EnergyButton {
  constructor(type) { // type: 'add' | 'subtract'
    this.type = type;
    this.pressState = 'idle'; // 'idle' | 'pressing' | 'longpress_fired'
    this.pressTimer = null;
    this.LONGPRESS_DELAY = 500; // ms
  }

  onTouchStart() {
    if (this.pressState !== 'idle') return; // 防重入
    this.pressState = 'pressing';
    this.pressTimer = setTimeout(() => {
      this.pressState = 'longpress_fired';
      this.onLongPress();
    }, this.LONGPRESS_DELAY);
  }

  onTouchEnd() {
    clearTimeout(this.pressTimer);
    if (this.pressState === 'pressing') {
      // 长按未触发，执行单击
      this.pressState = 'idle';
      this.onClick();
    } else if (this.pressState === 'longpress_fired') {
      // 长按已触发，touchEnd 不再执行单击
      this.pressState = 'idle';
    }
  }

  onTouchCancel() {
    // 手指滑出按钮区域，取消所有操作
    clearTimeout(this.pressTimer);
    this.pressState = 'idle';
  }

  onClick() {
    // 单击：静默调整，无气泡
    const delta = this.type === 'add' ? 0.5 : -0.5;
    updateEnergy(delta); // 内部处理边界 clamp
    playFishAnimation(this.type); // 飞入/淡出动画
    // ❌ 禁止在此处触发气泡
  }

  onLongPress() {
    // 长按：进入二级入口
    if (this.type === 'add') {
      navigateTo('/pages/add-detail/index');
    } else {
      navigateTo('/pages/subtract-detail/index');
    }
  }
}
```

#### 2.3 精力值更新函数（带边界和触发判断）

```javascript
/**
 * 更新精力值的统一入口
 * @param {number} delta - 变化量（+0.5 或 -0.5，或二级入口的自定义量）
 * @param {string} [source] - 来源：'quick'（单击） | 'detail'（二级入口）
 * @param {object} [record] - 详细记录（二级入口时传入）
 */
async function updateEnergy(delta, source = 'quick', record = null) {
  const today = getTodayDateString();
  const prev = parseFloat(localStorage.getItem(`energy_current_${today}`) ?? 10);
  const next = clamp(prev + delta, 0, 10);

  // 1. 更新本地缓存（同步，保证离线可用）
  localStorage.setItem(`energy_current_${today}`, next);

  // 2. 异步写入云端 Supabase
  const supabase = getSupabaseClient();
  supabase.from('energy_records').insert({
    user_id: supabase.auth.user().id,
    timestamp: new Date().toISOString(),
    energy_score: next,
    previous_score: prev,
    score_change: delta,
    event_category: record?.category ?? null,
    event_detail: record?.detail ?? null,
    change_type: delta > 0 ? 'add' : 'subtract',
    source
  }).then(({ error }) => {
    if (error) console.warn('云端记录失败，已暂存本地', error);
  });

  // 3. 更新UI
  renderFishIcons(next);
  updateTopDisplay(next);

  // 4. 触发检查（单击来源时，延迟300ms后检查，避免动画期间弹窗）
  if (source === 'quick') {
    setTimeout(() => checkTriggers(next, prev), 300);
  } else {
    checkTriggers(next, prev);
  }
}
```

### 功能 3：盲盒回血（微行动推荐）

#### 3.1 触发条件与优先级（防冲突设计）

> 所有触发检查通过统一入口 `checkTriggers(currentScore, previousScore)` 处理，内部按优先级执行。

```javascript
/**
 * 统一触发检查入口
 * 优先级：严重透支 > 慢性疲劳 > 日常盲盒
 * 每次只触发一个，触发后设置冷却期
 */
async function checkTriggers(currentScore, previousScore) {
  // 全局气泡冷却：每次气泡关闭后，3秒内不再触发新气泡
  if (isBubbleCooldown()) return;

  const userId = supabase.auth.user().id;

  // 优先级1：严重透支预警
  if (currentScore <= 3) {
    const cooldownKey = `subtraction_cooldown_${getTodayDateString()}`;
    if (!localStorage.getItem(cooldownKey)) {
      localStorage.setItem(cooldownKey, 'true');
      triggerSubtractionPermission(userId, currentScore, 'severe_exhaustion');
      return; // 已触发，本次不再检查其他
    }
  }

  // 优先级2：突然骤降（比上次降低超过50%）
  if (previousScore && previousScore > 0) {
    const dropRate = (previousScore - currentScore) / previousScore;
    if (dropRate > 0.5 && currentScore > 3) { // >3 以免与优先级1重叠
      triggerSubtractionPermission(userId, currentScore, 'sudden_drop');
      return;
    }
  }

  // 优先级3：慢性疲劳（连续3天同一时段≤5条）
  if (currentScore <= 5) {
    const isChronicFatigue = await checkChronicFatigue(userId);
    if (isChronicFatigue) {
      triggerSubtractionPermission(userId, currentScore, 'chronic_fatigue');
      return;
    }
  }

  // 优先级4：普通盲盒推荐（精力≤5且非以上情况）
  if (currentScore <= 5) {
    triggerBlindBox(userId);
  }
}
```

#### 3.4 推荐算法

```javascript
/**
 * 加权随机抽取盲盒行动
 * 权重 = base_probability × 个人反馈修正系数
 * 连续换3次时，确保不重复推荐同一条目
 */
async function getRecommendedAction(userId, excludeIds = []) {
  const { data: actions } = await supabase.from('blind_box_actions').select('*');
  const { data: feedbacks } = await supabase.from('user_action_feedback').select('*').eq('user_id', userId);

  let filtered = actions.filter(a => !excludeIds.includes(a.id));
  if (filtered.length === 0) {
    // 所有行动都被换过了，重置排除列表
    filtered = actions;
  }

  const weighted = filtered.map(action => {
    const userFeedbacks = feedbacks.filter(f => f.action_id === action.id);
    let weight = action.base_probability;
    userFeedbacks.forEach(f => {
      if (f.feedback === 'positive') weight += 0.2;
      else if (f.feedback === 'negative') weight = Math.max(weight - 0.1, 0.05);
    });
    return { ...action, weight };
  });

  const totalWeight = weighted.reduce((sum, a) => sum + a.weight, 0);
  let rand = Math.random() * totalWeight;
  for (const action of weighted) {
    rand -= action.weight;
    if (rand <= 0) return action;
  }
  return weighted[weighted.length - 1]; // fallback
}
```

### 功能 7：AI 生成个人日报

#### 7.1 触发逻辑

```javascript
/**
 * 日报生成触发规则
 * 条件1：每天 21:00 自动生成（通过 Supabase Edge Function 定时触发器或外部 cron）
 * 条件2：用户手动点击【查看我的日报】时，若今日日报不存在，立即生成
 * 防止重复：检查 daily_reports 中当日记录是否已存在，存在则不重新生成
 */
async function generateDailyReport(userId, date) {
  const supabase = getSupabaseClient();
  
  // 前置检查：当日是否已有日报
  const { data: existing } = await supabase
    .from('daily_reports')
    .select('*')
    .eq('user_id', userId)
    .eq('report_date', date)
    .maybeSingle();
  if (existing) return existing;

  // 数据充分性检查：当日记录为0时不生成（避免空报告）
  const { data: records } = await supabase
    .from('energy_records')
    .select('*')
    .eq('user_id', userId)
    .gte('timestamp', `${date}T00:00:00Z`)
    .lt('timestamp', `${date}T23:59:59Z`);
  if (!records || records.length === 0) {
    return { content: null, reason: 'no_data' }; // 前端展示：“今天还没有记录，明天见~” 
  }

  const avgEnergy = records.reduce((sum, r) => sum + r.energy_score, 0) / records.length;
  const minRecord = records.reduce((min, r) => r.energy_score < min.energy_score ? r : min);
  const maxRecord = records.reduce((max, r) => r.energy_score > max.energy_score ? r : max);
  const events = records.filter(r => r.event_category).map(r => `${r.event_category}${r.event_detail ? `(${r.event_detail})` : ''}`);
  
  const { data: sessions } = await supabase
    .from('blind_box_sessions')
    .select('*')
    .eq('user_id', userId)
    .eq('session_date', date);
    
  const { data: baseline } = await supabase
    .from('user_baseline')
    .select('*')
    .eq('user_id', userId)
    .single();
  const { phase } = calculateTodayEnergy(baseline);

  const systemPrompt = `
你是「小鱼干」App 的陪伴猫咪，正在为用户生成今日精力日报。

【严格约束，不得违反】：
1. 只能描述以下真实发生的数据，绝对不能编造任何未发生的事件或情绪。
2. 不得使用“可能”“也许”“或许”等推测语气描述当日已发生的事实。
3. 不得在日报中对用户进行评判、打分或说教。
4. 第二段建议必须基于今日数据的实际规律，无规律则不写第二段。
5. 第三段周期预测仅在 cycle_phase 不为「未追踪」时出现。
6. 总字数控制在 120-180 字之间。

【语气要求】：温暖、陪伴感、像朋友写给朋友的观察日记，不说教、不催促。
  `.trim();

  const userPrompt = `
今天的数据（${date}）：
- 精力记录条数：${records.length} 条
- 平均精力值：${avgEnergy.toFixed(1)} 条
- 最低精力：${minRecord.energy_score} 条（${formatHour(minRecord.timestamp)}）
- 最高精力：${maxRecord.energy_score} 条（${formatHour(maxRecord.timestamp)}）
- 记录的消耗/补充事件：${events.length > 0 ? events.join('、') : '（无详细记录）'}
- 使用盲盒次数：${sessions.length} 次${sessions.length > 0 ? `，行动：${sessions.map(s => s.action_text).join('、')}` : ''}
- 当前周期阶段：${phase}

请生成今日精力日报。
  `.trim();

  // 调用 Edge Function 以保护 API Key
  const { data: { content } } = await supabase.functions.invoke('generate-daily-report', {
    body: { systemPrompt, userPrompt }
  });

  await supabase.from('daily_reports').insert({
    user_id: userId,
    report_date: date,
    report_content: content,
    generated_at: new Date().toISOString()
  });

  return { content };
}
```

```javascript
async function checkAchievements(userId) {
  const { data: achievements } = await supabase.from('achievements').select('*');
  const { data: unlocked } = await supabase.from('user_achievements').select('*').eq('user_id', userId);
  const unlockedIds = unlocked.map(a => a.achievement_id);
  const newlyUnlocked = [];

  for (const achievement of achievements) {
    if (unlockedIds.includes(achievement.id)) continue;
    const condition = achievement.trigger_condition;
    let shouldUnlock = false;

    switch (condition.type) {
      case 'consecutive_low_energy':
        shouldUnlock = await checkConsecutiveLowEnergy(userId, condition.days, condition.time_range, condition.threshold);
        break;
      case 'consecutive_records':
        shouldUnlock = await checkConsecutiveRecords(userId, condition.days);
        break;
      case 'daily_blind_box_count':
        shouldUnlock = await checkDailyBlindBoxCount(userId, condition.count);
        break;
      case 'daily_high_energy':
        shouldUnlock = await checkDailyHighEnergy(userId, condition.threshold, condition.min_records ?? 3);
        break;
    }

    if (shouldUnlock) {
      await supabase.from('user_achievements').insert({ user_id: userId, achievement_id: achievement.id });
      newlyUnlocked.push(achievement);
    }
  }

  if (newlyUnlocked.length > 0) {
    showAchievementQueue(newlyUnlocked);
  }
}
```

```javascript
async function recordEnergyKillerIfNeeded(userId, eventCategory, scoreChange) {
```

## 七、开发注意事项

### 7.1 隐私与数据安全

- **树洞对话**：默认仅存储本地 `localStorage`，用户在设置中显式开启后才同步云端。
- **敏感信息**：不收集真实姓名、手机号；云端仅关联 Supabase Auth UUID。
- **数据删除**：设置页提供【删除所有数据】，点击后需二次确认，删除所有云端和本地记录。
- **心理危机处理**：树洞 AI 检测到高危词汇时，输出固定危机干预文案 + 心理援助热线。

### 7.2 性能优化要点

- **动画性能**：小鱼干动画只用 `transform` + `opacity`，不修改 `left/top`。
- **Lottie 优化**：动画 JSON 文件放在 Supabase Storage，按需加载，不打包进 JS bundle。
- **数据库查询**：所有高频查询字段按索引要求建立。
- **本地缓存**：盲盒行动库、成就库等静态数据启动时一次性缓存到 `localStorage`，后续从 Supabase 定期刷新。
- **分页加载**：日报列表每页10条。

### 7.3 AI 调用成本控制

- **日报生成**：每日最多一次，生成后缓存，不重复调用。
- **树洞对话**：优先使用轻量模型；每日总量限制可酌情在前端或函数上加。
- **帮助问答**：常见问题使用本地固定文案。
- **token 限制**：prompt 控制在 500 tokens 以内。

### 7.4 用户体验细节

- **加载状态**：所有网络请求显示宠物摇尾巴动画，超过3秒显示“主人，网络有点慢，稍等一下~”。
- **错误处理**：网络失败时显示友好文案 “主人，网络好像有点问题，稍后再试试吧~”，允许重试。
- **离线保障**：精力记录先写 `localStorage`，联网后自动同步；同步失败时本地数据不丢失。
- **空状态**：历史日报列表为空时，展示 “还没有日报，今天记几条精力值就能生成了~”。

### 7.5 边界与异常处理

| 场景 | 处理方式 |
|------|---------|
| 精力值达到上限（10条） | + 按钮变灰不可点，猫咪说 “已经满能量啦，好好保持~” |
| 精力值达到下限（0条） | - 按钮变灰不可点，猫咪显示关怀气泡（减法许可优先级最高） |
| 基线测试中途关闭小程序 | 重新打开时询问：“上次的体检做到一半了，要继续吗？” → 恢复到中断的那一页 |
| AI API 调用失败 | 树洞使用预设固定回复降级（随机从10条预设回复中选一条），日报显示“今天的日报生成遇到了点麻烦，明天再试试~” |
| 用户无任何精力记录的日报请求 | 返回占位文案，不调用 AI |
| 周期数据中 last_period_start 为未来日期 | 忽略周期数据，P_cycle = 1.0，不报错崩溃 |
| 用户快速连续点击 +/- 按钮 | 允许连续响应，每次点击独立处理；但触发检查有300ms防抖，避免重复弹窗 |
